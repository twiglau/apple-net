# State

- state 是组件内部存储的动态数据，用来记录 “变化的东西”\
- 比如开关状态，表单输入，计数器值，切换按钮等。
- 一旦State改了，React会立即出发重新渲染组件，确保页面UI跟数据保持一致。

## 解释

- State(状态) 是 React 组件中用于管理动态数据的核心机制。简单说，它就像你组件的 “记忆体” -- 你写的组件需要记住什么数据，用户做了哪些操作，当前也卖弄是展开还是收起？
  
## 为什么React不允许你直接修改State?

- React 希望通过 setState 来统一管理组件状态的更新流程。
- 这样它才能做很多 “性能优化的魔法”： 比如批量更新，跳过没变化的地方，控制渲染顺序等。
- React 不仅是个 UI 框架，它更像是个贷调度的大脑。

## 原则

1. 组件级（本地）状态管理优先

- 在React里，最基本的状态管理方式就是组件自己管理自己的状态
- React设计哲学： 数字是单向流动的，从上往下传。
- 构建 “想树一样” 的组件结构，数据就像水一样从根往下流，而不是像蜘蛛网一样到处乱串。
