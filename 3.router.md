# React Router 的组成部分

- BrowserRouter: 包裹整个应用，启用 HTML 的地址管理
- Routes + Route: 控制路径该渲染哪个组件
- Link 和 NavLink: 代替 a 标签， 控制跳转而不刷新页面
- useParams,useNavigate, useSearchParams: 提供各种 Hook 帮你处理参数，导航，状态；

## react-router-dom 而不是 react-router?

- react-router 是核心库，react-router-dom 是专门为 Web 浏览器定制的版本。
- react-router 是 “路由系统的底层核心逻辑”

  > 1. 路由配置结构，比如 <Routes /> <Route />
  > 2. 路由匹配算法
  > 3. 一些通用的 hook, 比如 useRoutes

- react-router-dom: 专门给 “浏览器应用”的
  > 1. BrowserRouter 和 HashRouter
  > 2. Link, NavLink, useNavigate
  > 3. 默认依赖了 react-router.

## outlet

- 是一个 “内容插槽”或者“窗口”。 “你把当前匹配到子路由组件，插到我这儿来显示”
- 该组件是 react router 的核心，它是我们连接外层布局和内层页面的 “通道”。

1. `<Outlet />` 是嵌套路由的 “内容插槽”。它告诉 React Router: 把匹配到的子路由内容插在我这里。
2. 我们可以用它把公共的结构（比如 header, footer）和页面内容分开，让代码更清晰，可维护。
3. 多个 `<Outlet />` 配合多个布局组件，还可以让整个网站页面更有层次感，适应不同的业务场景，比如电商首页 VS 登录页。
4. 最重要一点，它让你的 React 项目更“工程化”。

## 创建路由配置文件 router.tsx

- 职责分离，结构更清晰

1. App.jsx 的职责应该是： 挂载根组件，接入路由，状态管理，国际化等 “入口级别”的东西。
2. 如果你把大坨路由写进 App.jsx,它很快就会变得又长又乱，维护起来分厂吃力。所以，把路由单独抽出去，
   router.jsx 专注定义页面结构和布局关系，让 App.jsx 专注提供上下文的组件挂载。这样，逻辑分工明确。

- React Router 通配符

1. React Router v6 引入了新的匹配机制，让我们能很方便地用 \* 匹配 “其他所有没匹配上的路径”。

```tsx
<Route path="*" element={<NoFoundPage />} />
```

2. Error Boundary,在 React Router v6 中，我们可以直接在路由配置里使用 errorElement,为每一个路由单独定义错误处理组件。

## Link vs NavLink

- Link: 代替 a 标签，控制跳转而不刷新页面.通过 React Router 在前端动态切换组件，保持也卖弄状态，
  性能更好，用户体验也更流畅。
- NavLink: Link 的增强版，可以自动添加 active 状态，常用于菜单导航，
  当前路由匹配时自动添加 active 状态，可以很方便地实现高亮效果。

## useNavigate

- React Router 提供一个编程式导航的 Hook，可以在 JavaScript 逻辑中实现 “跳转”。
- JS 里控制页面跳转的遥控器。

```lua
功能          写法
页面跳转      navigate('/path')
携带状态      navigate('/path', {state})
后退，前进    navigate(-1), navigate(1)
替换当前记录   navigate('/path', {replace: true})
路由守卫      useEffect + navigate()
```
