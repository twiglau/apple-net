# useReducer 范式，复杂状态管理

## 解决问题：组件里有多个 useState,每次更新状态都要小心不可变性，尤其当状态是深度嵌套对象或数组时，一次简单的更新可能需要你写一堆 ...展开运算符，代码冗长又容易出错。

- 这时候，useReducer 就派上用场了。它的核心思想很简单：把组件状态管理成一台小型状态机。通过 action 告诉它“我要做什么”，reducer 就 根据 action 类型返回新的状态。这样一来，所有状态更新都集中的 reducer, 逻辑清晰，可预测，尤其适合复杂的嵌套数据结构。

## 特点

1. 把所有状态集中到一个对象里
2. 通过 dispatch 派发动作，自动触发 reducer 逻辑
3. 状态变得更清晰，集中，可追踪。
4. 少 setState,或漏改 total,所有逻辑全部交给 reducer 来处理。

## reducer 与 action 的设计

- 对于 API 请求或购物车操作，先定义 action 类型

```lua
-- // API 请求有三个状态
FETCH_START: 请求开始
FETCH_SUCCESS: 请求成功
FETCH_ERROR: 请求失败

```

- reducer 内部用 switch(action.type) 来判断当前操作，根据不同的 action 返回新的状态。 优雅地把 "请求开始，成功，失败" 的状态更新集中在一个函数里，逻辑清晰可控。
